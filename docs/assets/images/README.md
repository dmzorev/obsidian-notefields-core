# Screenshot plan

Place final README and documentation screenshots in this directory. Use the default Obsidian theme unless a feature specifically depends on another theme. Capture one consistent vault, window width, zoom level, and accent color.

## Required images

### `notefields-overview.png`

- A note in Reading or Live Preview mode.
- Show Select, Multiselect, Icon Picker, Color Picker, and a collapsed Nested field together.
- Use realistic property names and values rather than `test` data.
- Keep the note title visible so the image reads as an actual workflow.
- Recommended crop: approximately 1400 × 800.

### `nested-fields.png`

- One expanded Nested object.
- Include a nested object, a list, a checkbox, an empty value, and the **Add property** action.
- Keep depth to two or three levels so structure remains readable.
- Recommended crop: approximately 1200 × 850.

### `option-collections.png`

- The editor for one shared Value Option collection.
- Show stored values, display titles, icons, and colors.
- Use a useful set such as project status or priority.
- Recommended crop: approximately 1200 × 750.

### `bases-fields.png`

- A Bases table with Select, Multiselect, Nested, Icon, and Color columns.
- Show several rows and one active editor.
- Choose column widths that demonstrate compact overflow behavior without clipping the screenshot itself.
- Recommended crop: approximately 1600 × 750.

## Optional image

### `mobile-properties.png`

Add this after the main four images if mobile support becomes a release focus. Show the same Select or Multiselect workflow in Obsidian Mobile. It is useful supporting evidence, but not necessary for the first README because it does not introduce a distinct capability.

## Export guidance

- Prefer PNG at 2× density for crisp Obsidian UI text.
- Remove personal paths, vault names, and unrelated notes.
- Avoid large empty areas around the relevant UI.
- Use the same sample data across all images to make the set feel intentional.
